# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A **Tabby terminal plugin** adding a persistent MobaXterm-style left sidebar plus a bottom server-stats bar. The sidebar is an always-mounted vertical icon rail with three inner tabs and one action button:

- **Sessions** — saved SSH profiles (`ProfilesService`), grouped (`config.store.groups`); click to launch a connected tab. Shows per-profile live-connection dots (`logic/sessionStatus.ts`).
- **SFTP** — a **self-built file list** (`sftpFileList.component.ts`) bound to the active SSH tab's SFTP channel, live-following the shell cwd with a pin/follow toggle. (Originally reused Tabby's `SFTPPanelComponent`; too rigid to extend, so it was replaced.)
- **Macros** — saved keystroke/command macros (`macrosTab.component.ts`); also reachable as a global hotkey **palette overlay** (`Ctrl-Space`).
- **tmux action** (rail button at the bottom — an action, not a tab) — enters tmux control-mode on the active SSH tab; rendered only when the tabby-tmux plugin is present and the tab supports it.

Plus two surfaces outside the rail: a **bottom status bar** (`serverStatsBar.component.ts`) with live CPU/mem/disk/uptime/sessions for the active SSH host, and a unified **settings tab** (`mobaxSettingsTab.component.ts`, id `mobax-sidebar`) under Tabby's Settings — rail-item toggles (Sessions/SFTP/Macros/tmux), status-bar on/off + poll interval, and the **vault auto-unlock** UI (`autoUnlockSettingsTab.component.ts`) embedded as a section.

Design philosophy: **reuse Tabby's built-in parts where practical** (sidebar/status-bar DOM mounting, profiles, session launching, `SettingsTabProvider` + `HotkeyProvider`); the SFTP file list is the main exception — fully self-built so it can support multi-select, drag-to-move, and keyboard navigation.

```bash

pnpm build                # webpack production -> dist/index.js
pnpm watch                # webpack --mode development --watch
pnpm test                 # vitest run — include is src/**/*.spec.ts (today every spec lives in src/logic/)
pnpm exec vitest run src/logic/follow.spec.ts   # single test file
pnpm lint                 # oxlint && eslint .  (oxlint must pass first; fail-fast)
pnpm lint:fix
pnpm format               # prettier --write .  (do NOT run casually — reformats the whole repo; verify via lint/tsc/build instead)
```

- `pnpm build` runs webpack only, emitting `dist/index.js`. For a throwaway type-check, run webpack or `pnpm exec tsc --noEmit` directly.

### Dev-load into Tabby

The build output must be linked into Tabby's user plugins dir, then Tabby **fully restarted** (quit and relaunch) to load it — there is no working renderer-reload shortcut in the user's Tabby (Ctrl+Shift+R does nothing), so never tell the user to "reload"; say "restart". A restart loads the **last-built** `dist/index.js`, so rebuild after editing and confirm the artifact actually contains your change before debugging contradictory behaviour. One-time junction (Windows):

```powershell
New-Item -ItemType Junction -Path "$env:APPDATA\tabby\plugins\node_modules\tabby-mobax-suite" -Target "<repo path>"
```

**Never auto-launch/restart Tabby — the user runs it.** UI behaviour can only be verified by the user restarting and reporting back (no automated e2e).

## Build & runtime model (read before changing the build)

- Entry is a default-exported `NgModule` in `src/index.ts`; `main` = `dist/index.js`.
- webpack: `target: 'node'`, `libraryTarget: 'commonjs2'`, **ts-loader only**. Externals: `tabby-core`/`tabby-terminal`/`tabby-ssh`/`tabby-settings`, every `@angular/*` (incl. `@angular/cdk/*`), `@ng-bootstrap/ng-bootstrap`, `rxjs`, `electron`, and `@electron/*` — at runtime `require(...)` resolves to the **host Tabby's** modules. The pinned dep versions only provide build-time typings + externals consistency; **the host owns the actual instances, so DI tokens are shared with the host.**
- Build deps are pinned to the latest **published** `tabby-*` npm packages — currently Angular `15.2.6`, rxjs `7.5.7`, tabby-\* `1.0.231-nightly.0`. Tabby publishes its plugin API to npm only as *nightly* builds: `1.0.231-nightly.0` is npm `latest`, and there is **no** package matching the desktop app's release number (the app is on `1.0.234`, but `tabby-core@1.0.234` is a 404 on npm). These pins only supply build-time types + webpack externals; the running app owns the real instances, so the app version and the pin need not match. A genuinely incompatible (e.g. major Angular) version breaks Angular DI at plugin load — re-verify before bumping, and only bump to a version that actually exists on npm.
- Components use **inline `template`/`styles`** so webpack needs no pug/scss loaders. Every sidebar / status-bar / palette / settings component is our own; the one host component we instantiate is Tabby's `<checkbox>` in the macro palette's "모든 매크로" toggle. To make it resolve in our JIT templates, the NgModule `imports` include **tabby-core's default-exported module** (`AppModule`, which `exports` `<checkbox>`/`<toggle>`/… and declares no `providers`, so no duplication) — the same way first-party plugins like `tabby-settings` pull those components in.
- `src/index.ts` calls `installVaultAutoUnlock()` **and** `injectThemeTokens()` **at module require time** (before the NgModule declaration and Angular bootstrap): the vault patch must beat `VaultService` construction (see the vault gotcha below), and the `:root` color tokens (`--mobax-*`, `src/themeTokens.ts`) must exist before any component JIT-compiles so usage sites need no fallback literal.

## Architecture

**Bootstrap (`src/index.ts`).** The NgModule registers three multi-providers — `ConfigProvider` (`SidebarConfigProvider`), `HotkeyProvider` (`MacroHotkeyProvider`), `SettingsTabProvider` (`MobaxSettingsTabProvider`; `AutoUnlockSettingsTabProvider` is no longer registered — its component is embedded inside the unified tab) — plus services `SidebarService`, `StatusBarService`, `MacroPaletteService`, `TransferService`, and the eager `SidebarBootstrap`. `SidebarBootstrap` waits `AppService.ready$`, then (after a 1s `setTimeout`) calls `sidebar.initialize()`, `statusBar.initialize()`, `palette.mount()`; subscribes `hotkeys.hotkey$` so `mobax-macros:open` toggles the palette — the palette hotkey is gated when the Macros tab is disabled (`mobaxSidebar.tabs.macros === false`), showing a "매크로 탭이 꺼져 있습니다" notice instead of opening; and calls `setNotifier(...)` to wire the vault notifier once `NotificationsService` exists.

**Sidebar + tabs.**
- `src/services/sidebar.service.ts` — **the fragile coupling to Tabby's DOM** for the sidebar. Injects `SidebarHostComponent` into `<app-root>` via `ComponentFactoryResolver` + `ApplicationRef.attachView` + `appRoot.insertBefore`, flips `app-root` to `flex-row`, and fixes the deeper content column's width (`dom/contentColumn.ts`). Also holds the per-tab SFTP follow-pin (`getFollowPin`/`setFollowPin`). Mirrors `tsukasagenesis/tabby-ssh-sidebar`.
- `src/components/sidebarHost.component.ts` — vertical icon rail (Sessions / SFTP / Macros + a bottom tmux action) + collapsible content pane + resize handle; persists width and collapsed state. Clicking the active icon collapses the pane (rail-only); clicking it again or another icon expands. Subscribes `AppService.activeTabChange$` (and the split's `focusChanged$`) to drive the inner tab per active tab from an in-memory `WeakMap` keyed on the focused leaf (`sidebarTabContext`); first activation defaults to SFTP for an SSH tab / Sessions otherwise. Manual rail clicks record the choice; the memory is never persisted and never auto-expands a collapsed pane.
- `src/components/sessionsTab.component.ts` — SSH profiles from `ProfilesService`, grouped (group ids → names via `config.store.groups`, collapse state in `mobaxSidebar.expandedGroups` via `logic/groupCollapse.ts`); click → `launchProfile`. Live-connection dots via `logic/sessionStatus.ts`.
- `src/components/sftpTab.component.ts` — the SFTP tab shell: our breadcrumb + toolbar header (up / refresh / new dir / upload / filter) and a bottom follow toggle; resolves the active SSH binding; follows shell cwd (OSC + /proc, gated by pin); hosts `<sftp-file-list>`.
- `src/components/sftpFileList.component.ts` — **the self-built SFTP file list**. `readdir` listing with sort/filter; single-click select, double-click open (folder → navigate, file → `editLocally`); multi-select (Ctrl/Shift click + rectangular marquee drag); keyboard nav (↑↓ / Home / End / PageUp-Down / Ctrl+A / Backspace=up / Esc / F5, plus Enter / F2 / Delete); native HTML5 drag — drop a row onto a folder/`..` = internal move, drop a single file outside the app (e.g. desktop) = export via `DragOutServer`; **native OS file/folder drop = upload** (onto a row or empty area); cut/paste = move (Ctrl+X / Ctrl+V — there is **no copy**); inline rename; row + empty-area context menus.

**Macros.**
- `src/components/macrosTab.component.ts` — sidebar tab to list / create / edit / delete / reorder macros (CDK reorder); persists `config.store.mobaxMacros.list` (via `Object.assign` — ConfigProxy). Each macro carries an optional `profileId` (scope): absent/null = global, otherwise bound to one saved profile. The tab's default view shows `global ∪ active-profile` (via `logic/macro.ts` `macrosForProfile`, active id from `activeProfileId`) with a "모든 매크로" toggle to show every scope; reordering a filtered view maps back onto the full list via `applyVisibleReorder`. The editor has a 적용 대상 selector (전역 / profile, populated from `ProfilesService`); new macros default to the active session's profile. Also **imports/exports** macros + variables together as a version-1 JSON file (`{ version: 1, macros, variables }`) via Electron save/open dialogs (`@electron/remote` + Node `fs`), through the pure `logic/macroIO.ts`: export = `buildExport` + `serializeExport`; import = `parseImport` (validate/sanitize) → `resolveProfileScopes` (keep an imported macro's `profileId` only if that profile exists locally, else fall back to global) → `countConflicts` → `applyImport`, with the conflict choice surfaced as 전체 교체 / 병합·덮어쓰기 / 병합·건너뛰기 (replace / merge-overwrite / merge-skip; imported items always get a fresh id).
- `src/components/macroPalette.component.ts` + `src/services/macroPalette.service.ts` — a fixed-position overlay palette mounted to `<app-root>` (mirrors `SidebarService`); search / arrow-nav / Enter; toggled by the `mobax-macros:open` hotkey. Scoped to the active session the same way (`macrosForProfile`) and shows a scope chip. CJK IME: `COMPOSITION_BUFFER_MODE` is disabled so search filters mid-composition.
- `src/services/macroRunner.service.ts` — resolves the focused terminal leaf (`focusedLeaf` + `isTerminalLeaf`: leaf has `sendInput` **and** `session.open === true`), then runs the pure `runMacro` (`logic/macro.ts`); shows a notice if no terminal is focused.
- `src/macroHotkeyProvider.ts` — registers the single `mobax-macros:open` hotkey (default `Ctrl-Space`).

**Variables.**
- Global, plain-text variables (`config.store.mobaxVariables.list`, items `{ id, name, value }`); `${name}` tokens in macro **command** steps are substituted at run time. Substitution happens once, in `MacroRunnerService.run` (the shared palette + sidebar run path), via the pure `logic/variables.ts` (`buildVarMap`/`resolveMacroSteps`/`substituteVariables`/`validVarName`). Undefined tokens are left literal; substitution is single-pass (a value's own `${...}` is not re-expanded). Managed in `src/components/variablesTab.component.ts` (`<variables-tab>`), shown inside the **Macros** sidebar tab as a "변수" mode (toolbar button toggles list ↔ edit ↔ variables) — it lives with macros because that's the only place variables are used. The macro editor also shows clickable name chips that insert `${name}` at the caret.

**Status bar.**
- `src/services/statusBar.service.ts` — injects `ServerStatsBarComponent` at the bottom of Tabby's content column (`findContentColumn`); gated by `mobaxStatusBar.enabled`. See the absolute-positioned reflow gotcha.
- `src/components/serverStatsBar.component.ts` — per-active-SSH-host stats: CPU (with a smooth gradient-filled sparkline area chart of recent samples — SVG geometry in the pure `logic/sparkline.ts`: `sparkPoints` maps a 0..100 series into a right-anchored sliding window, `smoothPath` draws a Catmull-Rom curve, `areaPath` closes it for the fill), memory, disk, uptime, user + session count, and an OS-distro logo. Polls every `mobaxStatusBar.intervalSeconds` (default 3, clamped to ≥1). Keeps per-session history so switching tabs shows an instant snapshot.
- `src/ssh/statsExec.ts` — runs a one-shot command on the active SSH connection via a dedicated russh **session channel** (`ssh.openSessionChannel()` → `activateChannel()` → `requestExec()`), accumulating until the `@@END` sentinel. `buildStatsCommand`/`parseStats` live in `logic/serverStats.ts` (procfs/`df`/`os-release` parsing).
- `src/ssh/tmuxProbe.ts` — a one-shot `command -v tmux` probe over the same exec path; returns `true` / `false` / `null`.

**Settings tab.**
- `src/mobaxSettingsTabProvider.ts` + `src/components/mobaxSettingsTab.component.ts` — one Tabby settings tab (`mobax-sidebar`) with three sections: rail-item toggles (`mobaxSidebar.tabs`), status-bar enable + poll interval (`mobaxStatusBar`), and the embedded `<auto-unlock-settings-tab>`. All writes mutate config leaves + `config.save()`; the sidebar/status-bar/rail react live via `ConfigService.changed$`.
- Disabled rail items send **no server traffic**: turning off the status bar destroys `ServerStatsBarComponent` (stops stats exec polling); the tmux toggle short-circuits `updateTmuxButtonVisibility` before the `command -v tmux` probe; SFTP off un-renders the SFTP tab (no readdir/`/proc`). When all four `tabs` flags are off, `SidebarService` removes the whole sidebar. Decision logic is the unit-tested `logic/tabsConfig.ts` (`enabledInnerTabs`/`anyRailItemEnabled`/`resolveActiveInnerTab`).

**Vault auto-unlock.**
- `src/services/vaultAutoUnlock.ts` — `installVaultAutoUnlock()` monkeypatches `VaultService.prototype.getPassphrase` / `decrypt` at require time; supplies the stored passphrase when `decideUnlock` says so, and on a wrong **supplied** passphrase self-disables (`lastFailed`), clears the store, and notifies.
- `src/autoUnlockStore.ts` — DPAPI-encrypted passphrase persisted in `localStorage['mobaxAutoUnlock']` via Electron `safeStorage`, reached through `@electron/remote` `getBuiltin('safeStorage')`.
- `src/autoUnlockSettingsTabProvider.ts` + `src/components/autoUnlockSettingsTab.component.ts` — a Tabby settings tab (`mobax-auto-unlock`) to save / clear the passphrase (validated against the live vault before saving).

**SFTP transfer infrastructure.**
- `src/sftpOps.ts` — thin SFTP op wrappers: `downloadFile`, `uploadFiles`, `makeDir`, `renameEntry`, `moveEntry`, `removeEntry`, plus recursive `walkRemoteDir`.
- `src/fsTransfer.ts` — Node-fs local I/O: `FsFileUpload` / `FsFileDownload` (stream adapters to Tabby's FileUpload/FileDownload interface) and `walkLocalDir` (parent-first dirs, skips + counts symlinks).
- `src/transferWrap.ts` — wraps Tabby's FileUpload/FileDownload to report bytes to a `ProgressSink` and poll for cancel (throws `CancelledError`).
- `src/services/transfer.service.ts` — root singleton holding one `TransferProgressState`; drives the bottom progress strip + cancel (state machine in `logic/transferProgress.ts`).
- `src/sftpLocalEdit.ts` — `editLocally`: download to a temp file, open in the OS editor, re-upload, and re-`chmod` to the live mode (see the mode gotcha below).
- `src/sftpProcFollower.ts` — reads `/proc/<shellPid>/cwd` over the list's SFTP session (see the cwd gotcha below).
- `src/dragOutServer.ts` — root `@Injectable DragOutServer`: a localhost HTTP server that hands Chromium's `DownloadURL` drag-out a one-shot `/dl/<token>` URL and streams that file from SFTP at drop time (a `ResFileDownload` adapter mirrors `FsFileDownload` but writes to an `http.ServerResponse`). Drives the bottom strip; tickets self-expire after 60 s. See the drag-out gotcha below.

## Reused Tabby APIs and non-obvious gotchas (hard-won — do not relearn)

- **Literal `${...}` / `{` in an inline template breaks Angular JIT.** Components here use inline `template` backtick strings compiled by Angular's JIT at plugin load. Two distinct traps stack: (1) the JS backtick literal interprets `${x}` as its own interpolation — escape with `\${x}` to get the literal text; (2) even then, Angular's template parser reads a literal `{` as **ICU message** syntax and fails JIT with `Invalid ICU message. Missing '}'` / `unescaped "{"`. So never put a literal `${name}` (or any lone `{`) in template markup — bind it via interpolation from a component field instead (`tokenExample = '${name}'` → `<code>{{ tokenExample }}</code>`). Interpolation output is set as text and not re-parsed, so it's safe (this is why the macro-editor variable chips use `{{ chipLabel(n) }}`). Note JIT template errors are invisible to `pnpm build`/`tsc` (templates compile at runtime) — only loading the plugin in Tabby (a full restart) surfaces them.
- **Active tab:** subscribe `AppService.activeTabChange$` (not `activeTab$`). Top-level tabs are `SplitTabComponent` wrappers → unwrap with `getFocusedTab()`; also subscribe its `focusChanged$` to react to split-pane focus changes.
- **`SSHTabComponent` is not a public `tabby-ssh` export** in this version — do not import it for `instanceof`. Duck-type the leaf instead (`leaf.sshSession` truthy **and** `leaf.session.open === true`) via `resolveSSHBinding`. The SSH tab exposes `sshSession` (an `SSHSession`, has `openSFTP()` — the file list opens its own SFTP channel from it) and `session` (an `SSHShellSession`, has `.open`, `getWorkingDirectory()`, and `oscProcessor.cwdReported$`).
- **Self-built list, not the reused panel:** extending Tabby's `SFTPPanelComponent` (select-on-click, multi-select, drag-to-move) proved impractical — its single click is compiled in (folder→navigate, file→download) and its context-menu provider can't be borrowed (`SFTPContextMenuItemProvider` resolves to `null` from every plugin-reachable injector). So the list is fully our own; it opens its own SFTP channel via `sshSession.openSFTP()` and builds its own context menus with `platform.popupContextMenu`.
- **`ConfigService.store` is a `ConfigProxy`:** object-valued keys (`mobaxSidebar`, `mobaxStatusBar`, `mobaxMacros`) are **getter-only**; only leaf keys are writable. `config.store.mobaxSidebar = {...}` throws `"has only a getter"`. Mutate leaves instead: `Object.assign(config.store.mobaxSidebar, patch)` then `config.save()`.
- **Row drag = unified native HTML5 DnD (replaced CDK — do not reintroduce `cdkDrag` here).** CDK and native drag can't share a row: CDK's `DragRef` `preventDefault()`s `dragstart`, killing the native drag (so `DownloadURL` is never set). So the SFTP list dropped CDK entirely. A *selected* row is `[draggable]` (mirrors the old `cdkDragDisabled`: not `..`, not renaming, must be selected — so marquee-from-unselected still works). `onRowDragStart` sets an `application/x-mobax-move` marker and, for a lone non-dir file (when `DragOutServer.ready`), a `DownloadURL`. The **drop location** decides the action: onto a folder/`..` inside the list = internal move (`(drop)` → `performMove` → `sftp.rename`, skip rules in `planMoves`; target found via `dropTargetFromEvent` reading `data-name` off `event.target`); outside the app = export (the OS fetches the DownloadURL — our `(drop)` is not called). (CDK is still used by `macrosTab`/`sessionsTab`, so `DragDropModule` stays in `index.ts`.)
- **Three drag intents on one list, kept disjoint by `dataTransfer.types`.** (1) incoming OS file drop = *upload* (`hasOsFiles` → `'Files'`); (2) our row drag carrying `application/x-mobax-move` = *internal move* (`hasMoveMarker`); (3) the same row drag also carrying `DownloadURL` = *export*, consumed by the OS shell when dropped outside (never by our `(drop)`). `onDragOver`/`onDrop` branch on `hasOsFiles` vs `hasMoveMarker`, so the three never cross. The dropped OS file's local path comes from `resolveDroppedPath`: `webUtils.getPathForFile` (Electron 30+, supported API) with a `File.path` fallback (current Tabby = Electron 28), reached via `window.nodeRequire('electron')` since webpack doesn't bundle electron.
- **Drag-out (export) = Chromium `DownloadURL` + a localhost SFTP stream (`src/dragOutServer.ts`), NOT Electron `startDrag` — hard-won, do not relearn.** (a) `DownloadURL` carries **exactly one file**, so drag-export is single-file only; folders/multi keep the context-menu "다운로드". Real loose-file folder/multi drag-export is **impossible** in this Electron: native OLE delayed-render has no `startDrag` hook (it only accepts `{files, icon}`, with no way to inject a custom IDataObject into Chromium's drag source), and pre-download-to-temp was rejected. (b) The OS shows **no progress UI** for DownloadURL drag-out, so `DragOutServer` drives our own bottom strip (`TransferService`) from the HTTP handler (works because `sftpTab` renders the strip via `changes$.pipe(auditTime(80))` + manual `detectChanges`, zone-independent). (c) Cancel needs a **one-shot token**: `res.destroy()` (how we abort) makes Chrome auto-retry the same URL ~5–6×, so the server serves each token once and `404`s any retry (no `start`, no strip) — otherwise one strip-cancel never sticks. (d) `dragstart` is synchronous (the browser captures `dataTransfer` when the handler returns), so the server is started eagerly in the file-list ctor (`void dragOut.ensureStarted()`) and the dragstart path is fully sync (`register`/`urlFor`/`setData`).
- **Remote exec IS available** (this corrects an earlier assumption that it wasn't): russh's `SSHSession.ssh` exposes `openSessionChannel()` → `activateChannel()` → `requestExec(cmd)` for a one-shot command on a dedicated channel (separate from the interactive shell). This is how the status bar + tmux probe run (`src/ssh/statsExec.ts`). **Detect completion by a printed `@@END` sentinel, NOT the channel's `eof$`/`closed$` events** — russh buffers per-channel events and replays them on subscribe (`ChannelEventBuffer`), so opening a fresh channel each poll can fire stale events early and truncate the trailing output. On timeout, return the buffer only if the sentinel was seen (else `null`, so the caller keeps its last good values rather than blanking).
- **No copy:** SFTP itself has no copy primitive, and copy is intentionally unimplemented — every move (drag-drop and cut/paste, Ctrl+X / Ctrl+V) is a `rename`. (This is now a product choice, not a hard limit: remote exec is reachable per the previous bullet, so a `cp`-over-exec copy is feasible if ever wanted.)
- **Status-bar reflow:** Tabby's tab-body panes are `position:absolute` filling the (relative) content column, so flex can't push a bottom bar below them. `StatusBarService` instead injects a stylesheet keyed on a `mobax-statusbar-host` marker class: every pane gets `height: calc(100% - 28px)` and `server-stats-bar` is absolutely pinned to the bottom 28px. Using a class (not inline styles) covers tab-body elements Tabby adds later (new tabs / splits).
- **Vault auto-unlock monkeypatch must run at require time.** `installVaultAutoUnlock()` is called in `src/index.ts` **before** the NgModule/Angular bootstrap because it patches `VaultService.prototype.{getPassphrase,decrypt}`, and Tabby constructs `VaultService` during bootstrap — patch the prototype first or the instance keeps the originals. The notifier is wired separately via `setNotifier` (NotificationsService isn't available at require time). The passphrase lives in `localStorage['mobaxAutoUnlock']`, DPAPI-encrypted by Electron `safeStorage` reached through `@electron/remote`'s `getBuiltin('safeStorage')` — **NOT** `require('electron')`, which throws `"process.mainModule.require is not a function"` in this bundled-main Tabby.
- **`SFTPSession.stat()` returns `mode: 0`** in this russh — russh copies the metadata via `Object.assign({}, md)`, which drops `permissions` (a non-enumerable getter), so `.mode` comes back `0`. To read a file's **live** permission bits, re-list its parent directory with `readdir()` instead (only the listing path reads `md.permissions` explicitly) and pick out the entry by name; the panel's own `item.mode` can be **stale** if the file was chmod'd elsewhere after the listing was cached. This is why `editLocally` (`src/sftpLocalEdit.ts`) reads the live mode via `readdir` and re-`chmod`s it after upload: Tabby's `upload()` writes `<file>.tabby-upload` + unlink + rename, recreating the inode and resetting the mode to the server umask, so the mode must be restored (the rsync `-p` pattern). Cost two debugging sessions — do not relearn.
- **cwd following is hybrid (OSC + /proc).** OSC (`cwdReported$`) stays authoritative; when it is silent, `SftpProcFollower` (`src/sftpProcFollower.ts`) reads `/proc/<shellPid>/cwd` over the list's SFTP session, triggered by `shell.output$` (debounced) + a 4s safety timer, deferring to OSC via an `oscFreshMs` window. The shell pid is resolved once by anchoring on the sftp-server's `readlink('/proc/self')` → its parent session sshd → the single child with a controlling tty (`src/logic/procCwd.ts`, unit-tested). Multiple shells on one multiplexed connection (`reuseSession`, default on) are ambiguous → safe fallback (no navigation); this persists when a closed tab leaves a still-alive orphan shell on the shared connection (verified: both bash stay state `S` with their ptys). Disabling the profile's "Reuse session" gives each tab its own connection, so `/proc` is always unambiguous. Requires Linux `/proc` + `CONFIG_PROC_CHILDREN`; degrades silently otherwise. Depends on russh SFTP being able to read size-0 procfs file contents — if that ever breaks, the exec-channel fallback is the escape hatch (now proven viable — see `statsExec.ts`).
- **Chroot / virtual-root path mismatch.** Some servers (Synology, jailed SSH) show the shell one namespace (e.g. `/volume3/web/share`) while the SFTP subsystem is chrooted to a subpath (e.g. `/web/share`), and no protocol query reveals the relationship. `logic/resolveSftpPath.ts` probes candidates (`pathCandidates` strips leading segments), learns the rewrite once a candidate lists OK (`derivePrefixMapping`), then applies it thereafter (`applyPrefixMapping`, idempotent) so cwd-follow + breadcrumbs land in the SFTP namespace.

## Conventions

- Persisted config (defaults in `src/config.ts`): `mobaxSidebar` (`collapsed` / `width` / `expandedGroups` / `tabs`), `mobaxStatusBar` (`enabled` / `intervalSeconds`), `mobaxMacros` (`list`), `mobaxVariables` (`list`), and a deep-merged `hotkeys['mobax-macros:open']` default (`Ctrl-Space`). `mobaxSidebar.tabs` (`sessions`/`sftp`/`macros`/`tmux`, all default true) are the settings-tab rail toggles — the sidebar is mounted whenever any is enabled and removed entirely when all four are off (no separate `visible` flag — `tabs` drives it). Object-valued keys are ConfigProxy getters — mutate via `Object.assign(config.store.<key>, patch)` + `config.save()`; nested leaves like `mobaxSidebar.tabs.sessions` are writable through the nested proxy (`Object.assign(config.store.mobaxSidebar.tabs, patch)`).
- **Not persisted (in-memory only):** the active inner tab and the SFTP follow toggle (`pinned`), both remembered per-tab keyed on the SSH leaf via `sidebarTabContext` — the inner tab in `sidebarHost.component.ts`, the follow-pin in `SidebarService` (`getFollowPin`/`setFollowPin`, a `WeakMap` so the entry drops when the tab is GC'd; it lives in the service because `sftpTab.component.ts` is `*ngIf`'d out and re-created on every inner-tab switch, which would otherwise reset it). Also in-memory only: the SFTP list's selection, keyboard focus, and cut clipboard; the vault passphrase cache; the per-session stats history; the macros tab's "모든 매크로" (show-all) toggle.
- All component subscriptions are torn down via `takeUntil(this.destroyed$)`. The injected sidebar and status-bar views are mounted once on `ready$` and stay mounted; collapsing only narrows the sidebar wrapper to the rail and `*ngIf`s out the content pane.
- Korean user-facing UI strings are intentional (user preference).
